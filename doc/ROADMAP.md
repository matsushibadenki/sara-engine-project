# **SARA Engine \- Evolution Roadmap**

## **ビジョン: 生体模倣型AIへのパラダイムシフト**

SARAエンジンは、現在のAI（PyTorch/TensorFlowなど）が依存する「密な行列演算」「誤差逆伝播法（BP）」「GPU」からの完全な脱却を目指します。純粋なスパイク駆動と局所学習（STDP）による、超低消費電力・高効率な次世代AIエンジンを実現するためのロードマップです。

## **1\. 既存DLフレームワーク（PyTorch/TensorFlow）から取り入れるべき「必須機能」のSNN的昇華**

### **A. 動的トポロジーと自動学習ルーティング (Autogradの代替)**
* **目標:** モジュールを繋ぐだけで、スパイクの伝播経路と局所的な学習則が自動的に構築・ルーティングされる機能の実装。

### **B. 直感的なモジュールAPI (nn.Moduleの代替)**
* **目標:** 視覚野、海馬などの生体脳の機能ブロックを、レゴブロックのように直感的に組み合わせて構築できる高位モジュールAPIの拡充。

### **C. 状態のシリアライゼーションと復元 (state\_dictの保存/読込)**
* **目標:** シナプス荷重、膜電位、発火閾値といった「動的な内部状態」を保存・ロードする機構の実装。

### **D. スパイク・ストリーム・データローダー (DataLoaderの代替)**
* **目標:** 連続データをリアルタイムでAERフォーマット等にエンコードし、非同期ストリームとして流し込むデータパイプラインの構築。

### **E. 実行環境の抽象化とエッジ最適化 (.to("device")の代替)**
* **目標:** マルチコアCPUへの並列最適化と、エッジマイコンや専用チップへ透過的にデプロイできるハードウェア抽象化層（HAL）の実装。

### **F. リアルタイム・ダイナミクス可視化 (TensorBoardの代替)**
* **目標:** ラスタープロット等をリアルタイムで監視する専用可視化ツール「Sara-Board」の開発。

## **2\. Transformerの代替となるための「理論的ブレイクスルー」開発**

### **A. 乗算と非線形演算に依存しない「Spike-driven Attention」**
* スパイクの発火タイミングや頻度のみで、情報の重要度を動的にルーティングするアテンション機構の確立。

### **B. 長距離依存性（Long-Range Dependencies）の保持**
* 海馬システムの統合等により、長大なコンテキストをシナプス構造として保持し続けるダイナミクスの実現。

### **C. 大域的最適化を実現する「局所学習則」**
* 予測符号化や報酬変調STDPにより、局所的なシナプス更新の連鎖だけで大域最適化を実現する学習理論の実装。

## **3\. 開発フェーズとマイルストーン**

### **Phase 1: コアアーキテクチャの確立とフレームワーク化 (完了)**
* [x] Rustコアによる非同期スパイク計算エンジンの基礎実装。
* [x] 誤差逆伝播（BP）と行列演算の完全排除ルールの徹底。
* [x] PyTorchライクな高位APIの実装（Moduleベースの直感的な構築）。
* [x] 膜電位やシナプス状態の完全なSave/Load機能の実装。
* [x] スパイクデータローダーとリアルタイム可視化ツールのプロトタイプ開発。

* [x] Spike-driven Attentionモデルのプロトタイプ実装。
