# **SARA Engine 高精度化ロードマップ**

## **フェーズ 1: 基盤の堅牢化 (Current \- Short-term)**

「動くもの」から「壊れないもの」への脱皮。

* \[x\] **ディレクトリ階層化**: data/raw, interim, processed の分離と管理。  
* \[x\] **統合CLI (sara\_cli.py)**: パイプラインの一元管理。  
* \[ \] **ユニットテストの拡充**: tests/ 内の全モジュールに対する自動テストカバレッジの向上。  
* \[ \] **記憶の刈り込み (Synaptic Pruning)**: 重みの低い不要な記憶を自動削除し、メモリ使用量を一定に保つ機能。

## **フェーズ 2: 精度と一般化の向上 (Mid-term)**

行列演算を使わず、SNN特有の性質でLLMに匹敵する「賢さ」を実現。

* \[ \] **SDR Overlap推論**: 完全一致でなくても、SDRのビットが一定割合重なれば連想を起動するアルゴリズムの実装。  
* \[ \] **階層的特徴抽出 (Hierarchical SDR)**:  
  * 視覚：エッジ（低次）→ 図形（中次）→ 物体（高次）  
  * 言語：文字（低次）→ 単語（中次）→ 文脈（高次）  
* \[ \] **マルチソース統合**: PDF, Web, CSV, 画像, 音声を一つの統合SNN空間で同時に学習する「共通言語空間」の形成。

## **フェーズ 3: 商用デプロイ・エコシステム (Long-term)**

外部との接続と実社会での運用。

* \[ \] **CAPI / Rust SDK**: 他の言語（C++, Swift, Kotlin）からSARAを呼び出すためのインターフェース。  
* \[ \] **エッジ最適化**: Raspberry Pi やモバイル端末でのリアルタイム学習・推論の安定化（GPU不要の証明）。  
* \[ \] **監視ダッシュボード**: モデルの「記憶の密度」や「学習の進捗」をリアルタイムで可視化するWeb UI。

## **性能目標 (KPI)**

* **推論速度**: CPU単体で1トークンあたり10ms以内。  
* **メモリ効率**: 100万トークンの知識を 2GB 以内のRAMで保持。  
* **学習効率**: 誤差逆伝播法を用いた従来モデルの1/10以下のデータ量での特定ドメイン適応。
